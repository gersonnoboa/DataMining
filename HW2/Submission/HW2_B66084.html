Exercise 1

1. What is the data about?

The data has information regarding work situation of people and the salaries they earn. This data has people from different countries, occupations and field of work (like in state agencies, private, self employed and similar). Aside from basic information such as age, gender and education level, it also has, for some reason, the job satisfaction they're experiencing in their current workplace.

2. What are different features and their types (discrete, continuous etc, range of values, possible meaning of the features)?

  1. X: An auto-increment column. Type numeric discrete. Range of values: 0 to 32560. Meaning: just an autoincrement value.
  2. Age: Type numeric continuous. Range of values: 17 to 90. Meaning: the age of the employee.
  3. Workclass: Type text, unorder qualitative so it can't be discrete or continuous. Can be Federal-gov, Local-gov, Never-worked, Private, Self-emp-inc, Self-emp-not-inc, State-gov or Without-pay. Meaning: the type of work in which the person is involved, be it for the government, private, self employed, etc.
  4. Education: Type text, unordered qualitative so it can't be discrete or continuous. Can range from all school and high school levels to levels in higher education, such as masters, doctors, etc. Meaning: the last level of education that the person has.
  5. Occupation: Type text, unordered qualitative so it can't be discrete or continuous. Can have several values related to the type of work that the person is currently doing. Meaning: The short description of the work that the person is doing, like sales, tech support, armed forces, etc.
  6. Capital gain: Type numeric discrete. Range of values: 0 to 99999. Meaning: probably can be the gain that the properties of the person experienced in the last year. It is safe to say that if this value has a number greater than 0, then the field Capital loss must have 0.
  7. Capital loss: Type numeric discrete. Range of values: 0 to 99999. Meaning: probably can be the loss that the properties of the person experienced in the last year. It is safe to say that if this value has a number greater than 0, then the field Capital gain must have 0.
  8. Native country: Type text, unordered qualitative so it can't be discrete or continuous. Can have values from 42 different countries. Meaning: the country where the person was born.
  9. Salaries: Type numeric discrete. Range of values: 38.3412463124841 to 140000. Meaning: the salaries that the person makes per year.
  10. Job satisfaction: Type numeric discrete. Range of values: 0 to 15. Meaning: in a scale from 0 to 15, the level of job satisfaction that the person is currently experiencing.
  11. Male: Type boolean. Range of values: 1, which can be interpreted as true, or NA, which can be interpreted as false. Meaning: the gender of the person recorded is male. If this value is true, then the column Female is false.
  12. Female: Type boolean: Range of values: 1, which can be interpreted as true, or NA, which can be interpreted as false. Meaning: the gender of the person recorded is female. If this value is true, then the column Male is false.

3. How many rows are in the dataset?

32561 rows

Data cleaning

The tool used for cleaning the data was OpenRefine. The changes were the following:

  • Removed the column X
  • In column Age, one value -57 changed to 57. Reason: age can't be negative.
  • In column Age, one value 320 changed to 20. Reason: there's no natural way of living for 320 years, so it was probably a typing error in which the person wanted to type 20 and, because of the closeness of the keys, typed 320. Note that, because of this same fact, it could also be a case in which the person could have wanted to type 2 and typed the 3 by accident. I chose 20 because the information says that he is a high school grad and has a generic "other service" occupation, so it is more likely to be a 20 year old.
  • In column age, 97 values "?" changed to "NA." Reason: this will make it easier to process in R.
  • In column Workclass, ten values "privat" changed to "Private." Reason: there is another group of values that have the value Private, so probably this is an input error.
  • In column Workclass, all workclass values have an space at the beginning, they were removed. Reason: this was probably an error in the application that filled the database. Even though it changes nothing, it's better if the values are as error free as possible.
  • In column Workclass, 1836 values "?" changed to "NA." Reason: this will make it easier to process in R.
  • In column Native.country, 891 values "Unitedstates" changed to "United-States." Reason: other countries with spaces in their names have a dash instead of the space, so, to follow that convention, it is better to normalize this value to "United-States" with a dash present.
  • In column Occupation, 1843 values "?" changed to "NA." Reason: this will make it easier to process in R.
  • In column Native.country, 882 values "UnitedStates" changed to "United-States." Reason: other countries with spaces in their names have a dash instead of the space, so, to follow that convention, it is better to normalize this value to "United-States" with a dash present.
  • In column Native.country, 20 values "Hong" changed to "Hong-Kong." Reason: there is no country named Hong, but Hong Kong. Added dash to follow convention of dash instead of space.
  • In column Native.country, all contry values have a space at the beginning, they were removed. Reason: this was probably an error in the application that filled the database. Even though it changes nothing, it's better if the values are as error free as possible.
  • In column Native.country, 20 values "South" changed to "South-Korea." Reason: South is not a country, and there are no African countries present in any rows, so this study was probably not done in Africa. Hence, South Korea seems suitable than South Africa.
  • In column Native.country, 583 values "?" changed to "NA." Reason: this will make it easier to process in R.
  • In column Jobsatisfaction, one value "Very good" changed to "11." Reason: Probably a satisfied worker but doesn't think that his job satisfaction is excellent, which could award a 15, so a 11 might be a good number to translate this very weird mistake. Unfortunately, this is pure speculation because we don't know how the data was filled.
